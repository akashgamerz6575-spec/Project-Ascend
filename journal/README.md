# Engineering Journal

A system of daily logs and periodic reviews to enforce deliberate reflection, track velocity, and monitor long-term goals.

## Purpose
To turn day-to-day coding activities into structured learning logs, ensuring that technical struggles, bugs, and breakthroughs are documented and analyzed.

## Usage Conventions
- **Daily Logs:** Copied from `templates/daily-journal.md` and saved as `journal/YYYY/MM-Month/YYYY-MM-DD.md`.
- **Reviews:** Weekly reviews go to `journal/YYYY/Wxx.md`, monthly to `journal/YYYY/MM-Month/review.md`, and quarterly reviews to `journal/YYYY/Qx.md`.

## Contribution Guidelines for Future Self
1. Log daily. If a day is missed, resume on the next calendar day without backfilling generic statements.
2. Be honest about bottlenecks and knowledge gaps. This journal is a feedback loop, not a performance review.

## Directory Structure Example
```
journal/
└── 2026/
    ├── W01.md
    ├── Q1.md
    └── 07-July/
        ├── 2026-07-10.md
        └── review.md
```
