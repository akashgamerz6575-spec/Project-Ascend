# Repository Assets

Storage for diagrams, flowcharts, architectural drawings, and screenshots.

## Purpose
To centralize static media assets linked inside system designs, project documentation, or roadmaps, ensuring that no links break over time.

## Usage Conventions
- Save images as low-footprint PNG/WebP files.
- Place all SVGs and diagram definitions here.

## Contribution Guidelines for Future Self
1. Avoid committing massive raw images. Compress all images before committing.
2. Use descriptive, lower-case filenames separated by hyphens (e.g., `maintenance-app-architecture.png`).
