# DRY: Don't Repeat Yourself

"Every piece of knowledge must have a single, unambiguous, authoritative representation within a system."

## Practical Execution
- **Avoid Duplication:** Code duplication increases surface area for bugs and maintenance overhead.
- **Over-Abstraction Warning:** Don't apply DRY prematurely. Sometimes, duplicate code is better than the wrong abstraction (referred to as WET - Write Everything Twice before abstracting).
