# Production-Grade Projects

Detailed case studies, system design notes, and code architecture for completed and active applications.

## Purpose
To showcase software development capability through production-grade projects. Each project demonstrates architectural design, testing practices, database modeling, and deployment strategy.

## Usage Conventions
- Every project must maintain a dedicated folder inside `projects/` with its own case study documenting structure using `templates/project.md`.
- Store layout plans, mockups, or ERDs in `assets/`.

## Contribution Guidelines for Future Self
1. Do not push incomplete project readmes. Update requirements, database designs, and API tables synchronously with code changes.
2. Provide local deployment instructions using Docker to guarantee environment parity.
