# Project Ascend Agent Rules

## Git Commit & Contribution Strategy
1. **Granular Commits:** Do not squash daily session updates into a single commit. Create a separate, individual commit for each file created or modified during the session (e.g., a separate commit for each misconception tracker, notes page, journal log, and TIL file). This establishes a detailed, transparent activity history on GitHub.
2. **Backdate Commit Dates:** Always specify the correct author date corresponding to the learning session day (e.g., `--date="YYYY-MM-DDT22:00:00"`) when committing. This ensures that the GitHub contribution graph maps commits to the actual day the learning took place.
3. **Consistency Grid Formatting:** Keep the learning consistency grid in the root `README.md` formatted as an inline SVG with vector squares. Use `#ebedf0` for inactive squares and `#0e4429` (dark green) for active learning days to maintain high visual standard and compatibility across light and dark modes.
