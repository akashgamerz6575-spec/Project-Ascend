# Data Structures & Algorithms (DSA) Tracker

The registry of solved algorithm challenges, complexity analyses, and pattern summaries.

## Purpose
To move beyond rote problem-solving (e.g., memorizing solutions) and focus on identifying patterns, optimizing time/space complexity, and documenting boundary checks.

## Usage Conventions
- Catalog new problems under their specific paradigm folder:
  - `dsa/arrays/`
  - `dsa/strings/`
  - `dsa/graphs/`
  - `dsa/dynamic-programming/`
- Every logged problem must copy the structure defined in `templates/dsa-problem.md`.

## Contribution Guidelines for Future Self
1. Always analyze brute-force complexity before optimizing.
2. Clearly isolate spatial complexity (auxiliary space vs. input memory allocation).
3. Document edge cases (empty collections, limit values, integer overflows).
