# Today I Learned: Functions, Memory Execution & Call Stack

- **Function Stack Frames:** Every function call creates its own private execution workspace (stack frame) in memory.
- **Local Variable Lifetimes:** Local variables are allocated within the function's stack frame. When the function returns, its frame is popped and all local variables are automatically destroyed.
- **Pass by Value Mechanics:** Arguments passed by value are copied into a new memory location allocated inside the function's stack frame. Modifying the parameter has no effect on the caller's variable.
- **Pass by Reference Mechanics:** Arguments passed by reference act as aliases. The parameter shares the exact same memory address as the caller's variable, allowing direct modification.
- **Call Stack Control Flow:** The CPU manages function execution sequence using the call stack, which operates on Last In, First Out (LIFO) behavior. The last function called is always the first one to finish.
- **Recursion Memory Allocation:** Each recursive call is a separate function call that allocates a brand new stack frame. 
- **Stack Overflow:** Infinite recursion or too-deep recursion continuously piles up stack frames. Because no frame is popped, it exhausts stack memory, resulting in a program-terminating Stack Overflow.
