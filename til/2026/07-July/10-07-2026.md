# Today I Learned

- **Memory Isolation:** C++ variables own separate, isolated memory locations by default.
- **Copy vs. Link:** Value assignment (`a = b`) is a one-time copy of bits; it does not establish a persistent connection between variables.
- **References as Aliases:** A reference is not a new variable with independent memory; it is simply a compiler-level nickname (alias) for an existing variable's address.
- **Reference Binding:** A reference must be initialized when declared and cannot be reseated (reassigned to point to another variable) after creation.
- **Reassigning via References:** Assigning a value to a reference (e.g., `ref = other`) modifies the value of the original variable, rather than re-binding the reference.
- **Context-Sensitive Ampersand (`&`):** The `&` token declares a reference in a type signature (e.g., `int &r`), but acts as the address-of operator in an expression (e.g., `&val`).
- **Pointer Basics:** A pointer is a distinct variable that stores a memory address (e.g., `0x7ffebf8a91ac`) as its value.
- **Indirect Modification:** Dereferencing a pointer (`*ptr`) allows reading or writing directly to the memory address it holds.
- **Pointer Copies:** Pointers can point to copies of variables. Modifying the value at the pointer will only mutate the copy, not the original variable.
- **Memory Visualization:** Visualizing RAM addresses and layouts is superior to memorizing syntax for debugging pointers and references.
